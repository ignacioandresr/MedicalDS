Documentación de API REST

Opciones de las entidades Usuarios y Productos.
API Base: /api/v1

Usuarios

Ejemplo de uso: 
curl -X GET http://tuapi.com/api/v1/usuarios
Códigos de respuesta:
  • 200 OK - Éxito
  • 401 Unauthorized - No autenticado

Ejemplo de uso: 
curl -X POST http://tuapi.com/api/v1/usuarios \
-H "Content-Type: application/json" \
-d '{"nombre":"Juan","email":"juan@correo.com","password":"123456"}'
Códigos de respuesta:
  • 201 Created - Usuario creado
  • 400 Bad Request - Datos inválidos
  • 409 Conflict - Usuario ya existe

Ejemplo de uso: 
curl -X GET http://tuapi.com/api/v1/usuarios/1
Códigos de respuesta:
  • 200 OK - Éxito
  • 404 Not Found - Usuario no existe

Ejemplo de uso: 
curl -X PUT http://tuapi.com/api/v1/usuarios/1 \
-H "Content-Type: application/json" \
-d '{"nombre":"Juan Pérez"}'
Códigos de respuesta:
  • 200 OK - Usuario actualizado
  • 404 Not Found - Usuario no existe

Ejemplo de uso: 
curl -X DELETE http://tuapi.com/api/v1/usuarios/1
Códigos de respuesta:
  • 204 No Content - Usuario eliminado
  • 404 Not Found - Usuario no existe

Productos

Ejemplo de uso: 
curl -X GET http://tuapi.com/api/v1/productos
Códigos de respuesta:
  • 200 OK - Éxito
  • 401 Unauthorized - No autenticado

Ejemplo de uso: 
curl -X POST http://tuapi.com/api/v1/productos \
-H "Content-Type: application/json" \
-d '{"nombre":"Laptop","precio":1499.99,"stock":5}'
Códigos de respuesta:
  • 201 Created - Producto creado
  • 400 Bad Request - Datos inválidos

Ejemplo de uso: 
curl -X GET http://tuapi.com/api/v1/productos/1
Códigos de respuesta:
  • 200 OK - Éxito
  • 404 Not Found - Producto no existe

Ejemplo de uso: 
curl -X PUT http://tuapi.com/api/v1/productos/1 \
-H "Content-Type: application/json" \
-d '{"precio":1399.99}'
Códigos de respuesta:
  • 200 OK - Producto actualizado
  • 404 Not Found - Producto no existe

Ejemplo de uso: 
curl -X DELETE http://tuapi.com/api/v1/productos/1
Códigos de respuesta:
  • 204 No Content - Producto eliminado
  • 404 Not Found - Producto no existe
